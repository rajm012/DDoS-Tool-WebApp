How to Use the DDoS Tool

This guide will help you understand how to use the DDoS tool effectively. Below are explanations for each attack type and how to test them.

UDP Flood

Description: This attack floods the target with UDP packets.

Values to Input:

  • Target IPs: Enter the IP address(es) of the target(s).
  • Target Port: Specify the port to attack.
  • Duration: Set the duration of the attack in seconds.
  • Intensity: Define the number of packets per second.

Testing: Use Wireshark to monitor UDP traffic or check the terminal for sent packets.

SYN Flood

Description: This attack floods the target with SYN requests, overwhelming the target's connection queue.

Values to Input:

  • Target IPs: Enter the IP address(es) of the target(s).
  • Target Port: Specify the port to attack.
  • Duration: Set the duration of the attack in seconds.
  • Intensity: Define the number of SYN packets per second.

Testing: Use Wireshark to monitor SYN packets or check the terminal for sent packets.

HTTP Flood

Description: This attack floods the target with HTTP requests, overwhelming the web server.

Values to Input:

  • Target IPs: Enter the IP address(es) of the target(s).
  • Target Port: Specify the port to attack (usually 80 or 443).
  • Duration: Set the duration of the attack in seconds.
  • Intensity: Define the number of HTTP requests per second.

Testing: Use Wireshark to monitor HTTP traffic or check the terminal for sent requests.

ICMP Flood

Description: This attack floods the target with ICMP (ping) packets.

Values to Input:

  • Target IPs: Enter the IP address(es) of the target(s).
  • Duration: Set the duration of the attack in seconds.
  • Intensity: Define the number of ICMP packets per second.

Testing: Use Wireshark to monitor ICMP traffic or check the terminal for sent packets.

Slowloris

Description: This attack sends partial HTTP requests to keep connections open, exhausting the target's resources.

Values to Input:

  • Target IPs: Enter the IP address(es) of the target(s).
  • Target Port: Specify the port to attack (usually 80 or 443).
  • Duration: Set the duration of the attack in seconds.
  • Intensity: Define the number of concurrent connections.

Testing: Use Wireshark to monitor HTTP traffic or check the terminal for open connections.

DNS Amplification

Description: This attack exploits DNS servers to flood the target with amplified traffic.

Values to Input:

  • Target IPs: Enter the IP address(es) of the target(s).
  • Duration: Set the duration of the attack in seconds.
  • Intensity: Define the number of DNS queries per second.

Testing: Use Wireshark to monitor DNS traffic or check the terminal for sent queries.

Viewing Output

You can monitor the output of your attacks using the following tools:

  • Terminal: Check the logs in the terminal where the tool is running.
  • Wireshark: Use Wireshark to capture and analyze network traffic.
  • Logs Page: View detailed logs on the Logs page.
